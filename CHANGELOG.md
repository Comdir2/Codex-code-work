# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project uses [Semantic Versioning](https://semver.org/spec/v2.0.0.html) for documented releases.

## [1.0.0] - 2026-07-20

### Added

- Initial `code-work` skill contract for scoped software implementation.
- Repository-aware change planning and strict task-boundary rules.
- Root-cause workflow for defect investigation.
- Evidence-backed implementation slices and narrow-to-broad verification.
- Adaptive parallelism with bounded workers, exclusive write ownership, shared-state serialization, sequential fallback, and integrated verification.
- Engineering decision guidance for change strategy, test levels, reliable tests, diagnosis, parallel safety, and completion evidence.
- English and Russian project documentation, quickstart, realistic prompt examples, contribution guidance, support policy, and optional services description.
