# Support

Code Work is an open-source community project. Community support is provided on a best-effort basis without a guaranteed response time.

## Before requesting help

1. Read the [README](README.md) and [quickstart](docs/quickstart.md).
2. Confirm that Codex was restarted after installation.
3. Confirm that `code-work/SKILL.md` exists in the active Codex skills directory.
4. Reduce the problem to the smallest prompt and repository context that still demonstrates it.
5. Search [existing issues](https://github.com/Comdir2/Codex-code-work/issues).

## Open an issue

If the problem remains, open a [GitHub issue](https://github.com/Comdir2/Codex-code-work/issues/new) containing:

- the intended workflow and what occurred instead;
- the relevant prompt with secrets and private data removed;
- Codex surface or environment, operating system, and installation method;
- the skill version or commit being used;
- a minimal public reproduction when possible;
- validation output or error text as plain text, not only a screenshot.

Never include tokens, credentials, cookies, private repository contents, customer data, internal hostnames, or confidential logs in a public issue. If a safe public reproduction is impossible, describe only the sanitized behavior boundary.

## Supported topics

Community issues are appropriate for:

- installation and skill-discovery problems;
- contradictions or unclear behavior in the public skill instructions;
- reproducible documentation defects;
- proposals that improve scoped implementation or verification evidence.

General Codex product support, account billing, GitHub account problems, and repository-specific application debugging are outside this project's community support scope.

## Independent verification

Questions about read-only review, testing, diagnosis, or release readiness may fit [Code Verification](https://github.com/Comdir2/Codex-code-verification) better than this implementation skill.

## Optional professional help

For repository onboarding, custom workflows, training, or a defined implementation engagement, see [Services](SERVICES.md). Paid services are optional and do not reduce access to the MIT-licensed skill.
